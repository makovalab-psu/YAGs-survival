# Project Configuration

## Code Style Preferences

- Keep code concise and focused - no unused functions or dead code
- All imports at the top of files (PEP 8 style)
- Minimal comments - code should be self-explanatory where possible
- No verbose explanations unless asked

## Communication Style

- Be direct and concise in responses
- Focus on what changed and why, not lengthy explanations
- Only explain complex decisions or non-obvious choices

## Project Context

Working with Google Sheets data visualization:
- Fetching data from public Google Sheets and google drive stored text files
- Creating matplotlib visualizations
- Python-based workflow

## Code Quality
- Prefer list comprehensions over loops where readable
- Use type hints for function signatures
- Handle errors explicitly, no bare except clauses

## Output
- Save plots to `/output` directory
- Use descriptive filenames with timestamps
